# AI 查询示例

## 1. 管理员 admin

- `本周各部门工单数量是多少？`
- `列出所有 P1 未关闭工单。`
- `最近低满意度反馈有哪些？`
- `哪些部门超时工单最多？`

预期效果：

- 返回全量部门统计
- 可看到风险提示
- 可看到低满意度反馈

## 2. 主管 manager

- `本周各部门工单数量是多少？`
- `查询所有待反馈工单。`
- `最近已升级工单有哪些？`

## 3. 客服 customer_service

- `查询客服部待分派工单。`
- `今天新增了哪些投诉？`
- `查询 U1 或 U2 紧急投诉。`

## 4. 财务 finance

- `查询费用争议类未关闭工单。`
- `列出退款相关投诉。`
- `哪些费用争议工单已经超时？`

预期效果：

- 只能返回财务售后范围
- 对安全事件详情会拒绝或脱敏

## 5. 安全 safety

- `查询所有安全事件工单。`
- `列出 P1 未关闭工单。`
- `哪些安全工单已经升级？`

## 6. 运营 operation

- `查询司机服务类投诉。`
- `列出取消争议相关工单。`
- `哪些司机相关工单仍在处理中？`

## 7. 普通员工 employee

- `我的待处理工单有哪些？`
- `我负责的 P1 工单有哪些？`
- `我最近处理过哪些工单？`

禁止示例：

- `查询所有员工的待办工单。`
- `列出所有部门统计。`
- `帮我删除所有工单。`

预期效果：

- 第一类返回 `PERMISSION_DENIED`
- 写操作请求返回只读提示
